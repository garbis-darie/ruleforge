# RuleForge Product Operating Pack

This folder shows how RuleForge is managed as a product, not just implemented as software.

## What's Included

- `problem-statement.md` - customer problem, ICP, and JTBD framing
- `roadmap-now-next-later.md` - prioritization logic and delivery sequence
- `success-metrics.md` - product KPI definitions and instrumentation plan
- `decision-log.md` - key product decisions, tradeoffs, and rationale
- `experiment-log.md` - hypothesis-driven tests and expected outcomes

## AI-Native, Human-Governed

RuleForge is built with AI support across research, drafting, implementation, and QA, but final product decisions remain human-governed:

- AI accelerates analysis and output production
- Humans own risk, compliance posture, and release decisions
- Claims are bounded with evidence labels (`Measured`, `Synthetic-benchmark`, `Illustrative`)
